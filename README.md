# 🌳 ต้นไม้แห่งความหวัง (Tree of Hope)

เกมระดมความคิดแบบเรียลไทม์ 2 ฝั่ง (อาจารย์–ผู้เรียน)
ต้นไม้ปลอดใบเป็นพื้นหลัง ผู้เรียนทยอยตอบคำถามและวาง **ใบ → ดอก → ผล** บนต้นไม้ร่วมกัน
อาจารย์เห็นภาพรวมและควบคุมลำดับขั้น รองรับผู้เล่น 100+ คนพร้อมกัน

## ✨ คุณสมบัติ

- **2 ฝั่ง**: อาจารย์ (สร้างห้อง + ควบคุมขั้นตอน + ดูภาพรวม) และ ผู้เรียน (ตอบ + วาง)
- **หลายห้องเรียน** แยกตามรหัส 6 หลัก แต่ละห้องมีต้นไม้และคำถามของตัวเอง
- **ลำดับขั้นที่อาจารย์ควบคุม**: ตอบ→วาง สลับกันทีละขั้น (ใบ → ดอก → ผล)
- **เรียลไทม์**: เห็นรายการที่เพื่อนวางปรากฏขึ้นบนต้นไม้ทันที (Socket.IO)
- **ลากวาง** รองรับทั้งเมาส์และจอสัมผัส (pointer events)
- **เก็บข้อมูลถาวร** ใน SQLite
- **รองรับ 100+ ผู้เล่น** พร้อมกันด้วย room-based broadcast + WAL mode

## 🛠 เทคโนโลยี

- Next.js 14 (App Router) + TypeScript
- Custom Node server (`server.js`) รัน Next.js + Socket.IO บน HTTP server ตัวเดียวกัน
- Socket.IO (rooms แยกตามรหัสห้อง)
- better-sqlite3 (WAL mode)


```bash
npm run build

```

### เปิดให้ผู้เล่นจากอินเทอร์เน็ต (คลาวด์)

รันเซิร์ฟเวอร์ที่ port 3000 บนเครื่องคลาวด์ แล้วใช้ reverse proxy (Nginx) หรือ tunnel (เช่น cloudflared) เปิดให้เข้าผ่าน HTTPS

> หมายเหตุ: Socket.IO ต้องการ long-running Node process จึงต้องรันด้วย `npm start` บน VM/Container (ใช้บน serverless ไม่ได้)

## 🎮 วิธีเล่น

### ฝั่งอาจารย์

1. เข้าหน้าแรก → **ฉันเป็นอาจารย์ — สร้างห้อง**
2. ตั้งคำถาม 3 ข้อ (ใบ/ดอก/ผล) → สร้างห้อง
3. ได้ **รหัสห้อง** สำหรับแจกนักเรียน และ **ลิงก์อาจารย์** (มี token) สำหรับตัวเอง
4. ใช้ **PhaseController** เลื่อนขั้นทีละขั้น:
   - `ตอบ: ใบไม้` → `วาง: ใบไม้` → `ตอบ: ดอกไม้` → `วาง: ดอกไม้` → `ตอบ: ผลไม้` → `วาง: ผลไม้`
5. ดู **ProgressBoard** เพื่อติดตามว่าใครตอบ/วางถึงไหนแล้ว
6. คลิกที่รายการใด ๆ บนต้นไม้เพื่อดูคำตอบ/ลบ

### ฝั่งผู้เรียน

1. เข้าหน้าแรก → กรอก **รหัสห้อง** + **ชื่อ** → เข้าร่วม
2. ทำตามขั้นปัจจุบันที่อาจารย์เปิด:
   - ช่วง **ตอบ**: พิมพ์คำตอบ → ส่ง
   - ช่วง **วาง**: เลือก emoji จากถาด → กดค้างแล้วลากไปวางบนต้นไม้
3. เห็นต้นไม้ร่วมกันที่เติบโตขึ้นเรื่อย ๆ เมื่อเพื่อนวางรายการใหม่

```

├── lib/
│   ├── db.js                 # query ฐานข้อมูลทั้งหมด (CommonJS)
│   ├── store.js              # session in-memory (socket -> room)
│   ├── socket.js             # event handlers Socket.IO (CommonJS)
│   ├── socket-client.ts      # singleton client
│   └── types.ts              # shared types
├── app/
│   ├── page.tsx              # หน้าแรก
│   ├── join/page.tsx         # ผู้เรียนกรอกรหัส + ชื่อ
│   ├── play/[code]/page.tsx  # หน้าเกมผู้เรียน
│   ├── teach/[code]/page.tsx # หน้าอาจารย์
│   └── api/rooms/route.ts    # API สร้าง/ตรวจห้อง
├── components/
│   ├── TreeCanvas.tsx        # พื้นหลังต้นไม้ + เลเยอร์รายการ
│   ├── ItemTray.tsx          # ถาดเลือก emoji
│   ├── PhaseController.tsx   # ปุ่มควบคุมขั้นตอน
│   ├── ProgressBoard.tsx     # ตารางภาพรวม
```

## 🔄 ลำดับขั้น (Phase)

| # | Phase | คำอธิบาย |
|---|-------|----------|
| 1 | `answer_leaf` | ผู้เรียนพิมพ์คำตอบ "ใบไม้" |
| 2 | `place_leaf` | ผู้เรียนลากใบไม้ไปวางบนต้นไม้ |
| 3 | `answer_flower` | พิมพ์คำตอบ "ดอกไม้" |
| 4 | `place_flower` | ลากดอกไม้ไปวาง |
| 5 | `answer_fruit` | พิมพ์คำตอบ "ผลไม้" |
| 6 | `place_fruit` | ลากผลไม้ไปวาง → จบ |

อาจารย์เลื่อน phase ไป/กลับได้ตลอดเวลา และเห็นตัวเลขว่าแต่ละขั้นมีกี่คนทำเสร็จ

## ⚖️ หมายเหตุ

- ลิงก์อาจารย์ใช้ `teacher_token` ป้องกันนักเรียนเปิดหน้าควบคุม (ระดับห้องเรียน)
- หากต้องการขยายเป็นหลายเซิร์ฟเวอร์ ให้เพิ่ม `@socket.io/redis-adapter` และย้าย in-memory store ไป Redis
- ใบ/ดอก/ผล ใช้ emoji หากต้องการใช้ PNG วางไฟล์ใน `/public` แล้วแก้ `EMOJIS` ใน `lib/types.ts`
